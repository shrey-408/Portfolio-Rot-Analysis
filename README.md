# Agent Fatigue & Portfolio Rot Detector

**Collections analytics project for NBFC distressed loan portfolio management.**

Two silent drains most teams never model:
1. **Portfolio Rot** — Recovery probability decays non-linearly. There is a cliff. After it, accounts are nearly unrecoverable.
2. **Agent Fatigue** — Caseload above a threshold measurably drops conversion rates.

---

## Files

| File | Purpose |
|---|---|
| `data_generator.py` | Generates synthetic collections portfolio with realistic, systematic mess |
| `analysis.ipynb` | Full analysis: cleaning → decay curve → fatigue detection → reallocation → P&L sim → memo |
| `rule_engine.py` | Daily call prioritisation engine: scores accounts by EV × urgency, assigns offers, flags fatigued agents |
| `pl_simulator.py` | P&L simulation: compares three calling strategies (current, DPD-only, rule engine) over 90 days |
| `app.py` | Interactive Streamlit dashboard |
| `data/` | Generated by `data_generator.py` |

## Key findings

- Portfolio recovery cliff: **~Day 22** (logistic decay, inflection point)
- Agent fatigue threshold: **>60 accounts** (conversion drops ~30–40% relative)
- Data quality catch: **AGT017 CRM break at Day 45** (systematic null outcome pattern)
- Reallocation algorithm: greedy by **loan-type specialisation**, not just overall skill

---

## Design choices

- Data mess is **systematic**, not random: duplicate clusters on high-volume agents, future-dated entries at EOM, one broken CRM pipeline (AGT017)
- **Circularity acknowledged**: the cliff (Day 22) and fatigue threshold (60 accounts) are encoded in the data generator by design. The purpose is to validate the *detection method* — that the logistic curve fit and caseload-conversion analysis can accurately recover known parameters. In production, this same pipeline runs on real call logs and the cliff is discovered, not confirmed.
- Fatigue filter in `rule_engine.py` **flags** overloaded agents rather than silently dropping their accounts — the calling strategy decides whether to skip or deprioritise
- Reallocation routes by **loan-type skill** (a microfinance agent on a business loan is a wasted call)
- P&L simulator compares three strategies head-to-head: no sort (baseline), DPD-only, and EV × urgency rule engine
- Memo contains real numbers from the simulation output, not placeholders
